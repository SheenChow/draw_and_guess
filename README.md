# 你画我猜（双人版）

基于 **Next.js**、**React 19** 与 **Socket.io** 的双人实时「你画我猜」 Web 应用：画手与猜词使用**不同页面**，共享同一块画板；词语仅在服务端校验，猜词端看不到答案。

## 技术栈

- Next.js（App Router）、React 19、TypeScript
- Tailwind CSS v4
- 自定义 Node 入口 `server.mjs`：同一进程同时提供 Next 页面与 Socket.io（路径 `/socket.io/`）

## 环境要求

- **Node.js** 建议使用当前 LTS（例如 20.x），并与 Next.js / ESLint 要求兼容。

## 安装与运行

### 1. 安装依赖

```bash
npm install
```

### 2. 本地开发

```bash
npm run dev
```

默认在 **http://localhost:3002** 启动。浏览器打开首页即可创建房间并得到「画手」「猜词」两个链接。

### 3. 生产构建与启动

```bash
npm run build
npm start
```

同样默认监听端口 **3002**（见下方环境变量）。

### 4. 代码检查

```bash
npm run lint
```

## 如何使用（玩法）

1. 在一台已启动服务的电脑上打开首页，页面会生成一个**房间 ID**，并提供两个入口：
   - **我是画手**：进入作画页，可设置本局要猜的**词语**（仅你和服务器知道），在画布上绘画。
   - **我是猜手**：进入仅可查看的画布页，根据画面输入猜测。
2. 将**同一房间**下的两个链接分别发给两名玩家（或自己在两个浏览器标签中打开测试）。
3. 猜手提交词语后，由服务器与画手设定的词语比对（忽略中英文大小写与首尾空格）；猜对后双方会看到提示；**新一局**由画手点击发起。

**注意：** 所有实时数据依赖 WebSocket 连接到**同一台**运行 `server.mjs` 的服务器。另一台手机或电脑访问时，请使用该服务器的可访问地址（例如局域网 IP），不能使用与本机不同的部署实例。

### 局域网访问示例

本机 IP 为 `192.168.1.100`、端口为 `3002` 时，用手机浏览器打开：

```text
http://192.168.1.100:3002
```

在手机与电脑上分别从首页进入同一房间的「画手」「猜词」链接即可联机。

## 环境变量（可选）

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `PORT` | HTTP 端口 | `3002` |
| `HOSTNAME` | 传给 Next 的主机名 | `localhost` |
| `NODE_ENV` | `production` 时使用生产模式（由 `npm start` 设置） | — |

端口实际取值优先级为：**命令行参数**（例如 `npm run dev -- 3002` 或 `node server.mjs 3002`）→ **`PORT` 环境变量** → **默认 3002**。若终端里长期导出过 `PORT=3000`，仅用 `npm run dev` 仍会监听 3000；可改用 `npm run dev -- 3002`，或执行 `unset PORT` 后再启动。

## 目录结构（简要）

```text
├── server.mjs              # Node HTTP + Next + Socket.io
├── app/                    # Next.js App Router 页面
│   ├── page.tsx            # 首页（创建房间）
│   └── room/[roomId]/      # 画手 / 猜词页面
├── components/             # 画布与首页组件
├── lib/socket.ts           # 客户端 Socket 封装
└── types/game.ts           # 游戏相关类型
```

## 说明

- 房间与画布状态保存在**进程内存**中，重启服务后会清空；也不适合多实例无共享存储的横向扩展。若需持久化或多机部署，需自行接入 Redis 等并实现会话与房间同步。
